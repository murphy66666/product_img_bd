# AGENTS.md

## 项目背景

这是一个使用大模型生成商品图片的项目，采用前后端分离架构。前端项目在./frontend。后端backend作为前端的接口API支撑。

## 技术栈
- 后端：Python3 + FastAPI + Uvicorn + Langchain + UV
- 前端：Vue3 + Pinia + Ant Design Vue + UnoCSS
- 数据库：MySQL 8.x.x
- 缓存：Redis
- 目录监听：watchdog

## 工作规则
- **先读后写**：修改代码前，必须先完整阅读相关的项目结构和现有代码。
- **影响评估**：动手修改前，必须先说明会影响哪些文件或模块。
- **启动与测试**：修改完成后，必须给出明确的启动命令和测试步骤。
- **模块化原则**：不允许把所有功能堆砌在一个页面或一个巨大的文件中。
- **代码规范**：严禁把 HTML 写死在后端（如 Python/Java）的字符串中。
- **数据库约束**：不允许默认使用 SQLite，生产环境优先使用 MySQL。
- **高危操作限制**：涉及数据库结构修改时，必须给出完整的 SQL 语句；涉及删除文件、清空数据等操作时，必须有二次确认提示、操作日志记录以及回滚方案说明。

## 输出规范
- 保持原有的文件名和目录结构不被破坏。
- JSON 接口输出格式必须稳定，可供其他系统对接。

## 完成标准
每次任务完成后，必须按以下清单进行回复：
1. 修改了哪些文件？具体修改了什么内容？
2. 如何启动后端服务？如何启动前端项目？
3. 如何验证新功能是否生效？
4. 本次修改是否涉及数据库 SQL 变动？
5. 当前是否存在潜在风险？如果有，回滚方式是什么？
